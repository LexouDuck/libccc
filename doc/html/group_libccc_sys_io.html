Detailed Documentation¶

This header defines all the functions for OS-level input/output, read/write.

  • C ISO Standard: https://en.cppreference.com/w/c/io

Typedefs¶

typedef int t_fd

File descriptor integer type for libccc/sys/io functions.

Define a type for file descriptors (which is usually the default machine ‘int’)

typedef int t_io_error

This type represents an error code for a system call IO function (ie: a value for the ‘errno’ global variable)

  • C ISO Standard: https://en.cppreference.com/w/c/error/errno

  • C ISO Standard: https://en.cppreference.com/w/c/error/errno_macros

typedef unsigned int t_io_mode

This type represents a file access permissions bitflag value (like you would use with chmod)

typedef unsigned int t_io_open

This type represents the bitflag value which handles options for the open() function.

Global Functions¶

t_fd IO_Open(char const* filepath, t_io_open flags, t_io_mode mode)

Opens the file at ‘filepath’, with the given file open ‘flags’, and the given file access ‘mode’ permissions.

SYSCALL wrapper: int open(const char *pathname, int flags, mode_t mode);

Parameters:

filepath

The path of the file to open

flags

The flags with which to open the file (bitflag, can or bitwise OR’d, for example: OPEN_READONLY|OPEN_CREATE)

mode

The file access mode (permissions) with which to open the file

Returns:

the new file descriptor value for the opened file, or a non-zero error code (ie: an ‘errno’ value)

t_io_error IO_Close(t_fd fd)

Closes the given file descriptor ‘fd’, so that it no longer refers to any file, and may be reused.

SYSCALL wrapper: int close(int fd);

Parameters:

fd

The file descriptor to close

Returns:

0(OK) on success, or a non-zero error code (ie: an ‘errno’ value)

char* IO_GetError(int error_num)

Returns the error message (a string literal) corresponding to the given ‘errno’ error number.

t_bool IO_IsTerminal(t_fd fd)

Returns 1(TRUE) if the given file fd corresponds to a terminal/commandline rather than a file.

t_io_error IO_ChangeMode(char const* filepath, t_io_mode mode)

Changes the file access mode permissions for the file at ‘filepath’ to the new value ‘mode’.

SYSCALL wrapper: int chmod(const char *pathname, mode_t mode);

Parameters:

filepath

The path of the file whose access permissions should change

mode

The new file access mode permissions bitflag value to set

Returns:

0(OK) on success, or a non-zero error code (ie: an ‘errno’ value)

t_io_error IO_Read_File(t_fd const fd, char** a_file, t_size max)

Reads the contents of ‘fd’ and fill a data buffer with those bytes.

Reads the full contents of the file descriptor ‘fd’, and puts it into ‘a_file’.

Parameters:

fd

The file descriptor to read data from

a_file

The address of the buffer to fill: is allocated and filled with the data

max

The maximum amount of bytes to read from ‘fd’; if 0, replaced by (t_size)-1

Returns:

0(OK) if the stream was read successfully, or a non-zero error code (ie: an ‘errno’ value)

t_io_error IO_Read_Lines(t_fd const fd, char*** a_strarr)

Reads the contents of ‘fd’ and makes an array of strings, one for each line.

Reads the contents of the file descriptor ‘fd’, and puts that into an array of strings, one char pointer for each line (each ‘n’ newline char). Each ‘n’ character in the string are replaced by ‘0’ string terminators.

Parameters:

fd

The file descriptor to read data from

a_strarr

The address of the string array to fill: the top-level top-level pointer array is allocated and filled appropriately.

Returns:

0(OK) if the stream was read successfully, or a non-zero error code (ie: an ‘errno’ value)

int IO_Read_NextLine(t_fd const fd, char** a_line)

Reads the contents of the file descriptor ‘fd’ line-per-line.

This function will read ‘fd’ line-per-line as you call it repeatedly. This means that it will allocate and fill a buffer until reading ‘n’, at which point this buffer is returned via the ‘line’ char pointer arg. As such, this function will typically be called within a while loop.

Parameters:

fd

The file descriptor to read data from

a_line

The address of the string to allocate and fill with the current line.

Returns:

+1 (GNL_LINE) if a line of characters was successfully read 0 (GNL_END) if the end of the file was reached -1 (GNL_ERROR) if there’s an error

#define IO_BUFFER_SIZE  (2048)

Arbitrary buffer size for libccc/sys/io functions.

This is the arbitrary buffer size to be used by the reading functions. Raising this amount will lower the amount of function calls made to the ‘read’ function from unistd.h, resulting in possible speed improvments, depending on the amount of files to read, and their filesizes. It is also recommended to have this number be a power of 2, as it can be occasionally faster to manage arrays of such sizes on certain machines.
